# GWF 使用说明

[中文](../README.md) · [English intro](../README.en.md)

安装、初始化、在 Grok Build 里怎么用（详细说明仅中文）。

## 安装

```bash
npm install -g gwf-cli --registry https://registry.npmjs.org/
gwf -V
```

升级：

```bash
npm install -g gwf-cli@latest --registry https://registry.npmjs.org/
# 或 gwf upgrade
```

已 init 的项目升级 CLI 后建议：`gwf update`。

## 初始化

```bash
cd 你的项目
gwf init -u 你的名字
```

结束时会问是否开启（**直接回车 = 是**）：

1. 打开 Grok 自动加载工作流（不用每次 `/start`）
2. 提交前检查改动范围

```bash
gwf init -u 你的名字 -y
gwf init -u 你的名字 --no-automations
```

补开：`gwf enable-automations` / `gwf trust` / `gwf install-hooks`。

```bash
git add .gwf .grok AGENTS.md .gitignore
git commit -m "chore: init GWF workflow"
```

## 日常

```bash
cd 你的项目
grok
# 直接说需求
```

| 场景 | 你做什么 |
|------|----------|
| 小改 | 直接说 |
| 大改 | 确认任务与改动范围（scope） |
| 提交被拦 | 超出 scope；收窄改动或更新 scope |

**scope**：限制本次允许改的路径，一般由 AI 起草，你确认。

## 命令

| 命令 | 作用 |
|------|------|
| `gwf init -u <name>` | 初始化 |
| `gwf update` | 同步模板 |
| `gwf upgrade` | 升级 CLI |
| `gwf doctor` | 检查环境 |
| `gwf trust` | 仅自动加载 |
| `gwf enable-automations` | 自动加载 + 提交检查 |
| `gwf install-hooks` | 仅提交检查 |

## 排错

| 现象 | 处理 |
|------|------|
| 不认识 GWF | 项目根目录打开 Grok；`gwf trust` |
| 找不到 `gwf` | 重新全局安装，重开终端 |
| 镜像无新版 | `--registry https://registry.npmjs.org/` |
