# GWF 使用说明

面向日常用户：安装、初始化和在 Grok Build 里怎么用。  
（详细文档仅中文；仓库首页 [README](../README.md) 提供中英对照的简介。）

## 安装

```bash
npm install -g gwf-cli --registry https://registry.npmjs.org/
gwf -V
```

升级：

```bash
npm install -g gwf-cli@latest --registry https://registry.npmjs.org/
# 或 gwf upgrade
```

已 init 的项目在升级 CLI 后建议执行：`gwf update`。

## 初始化项目

```bash
cd 你的项目
gwf init -u 你的名字
```

结束时会问是否开启两项（**直接回车 = 是**）：

1. **打开 Grok 自动加载工作流**（不用每次 `/start`）
2. **提交前检查改动范围**（防止 AI 改到任务范围外还被 commit）

```bash
gwf init -u 你的名字 -y              # 不提问，全开
gwf init -u 你的名字 --no-automations  # 全不开
```

已 init 过要补开：`gwf enable-automations` 或 `gwf trust` / `gwf install-hooks`。

建议提交脚手架：

```bash
git add .gwf .grok AGENTS.md .gitignore
git commit -m "chore: init GWF workflow"
```

## 日常使用

```bash
cd 你的项目
grok
# 直接说需求
```

| 场景 | 你做什么 |
|------|----------|
| 小改 | 直接说即可 |
| 大改 | 确认是否建任务、确认改哪些目录（scope） |
| 提交被拦 | 改动超出本次任务范围；收窄改动或让 AI 更新 scope |

**scope**：限制「这次允许改哪些路径」，一般由 AI 起草，你确认，不必手写。

## 常用命令

| 命令 | 作用 |
|------|------|
| `gwf init -u <name>` | 初始化 |
| `gwf update` | 同步项目模板 |
| `gwf upgrade` | 升级全局 CLI |
| `gwf doctor` | 检查环境 |
| `gwf trust` | 仅开启 Grok 自动加载 |
| `gwf enable-automations` | 再次开启自动加载 + 提交检查 |
| `gwf install-hooks` | 仅安装提交检查 |

## 排错

| 现象 | 处理 |
|------|------|
| Grok 像不认识 GWF | 在项目根目录打开；执行 `gwf trust` |
| 找不到 `gwf` | 重新 `npm i -g gwf-cli`，重开终端 |
| 镜像没有最新版 | 加 `--registry https://registry.npmjs.org/` |
