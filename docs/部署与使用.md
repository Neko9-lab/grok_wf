# GWF 部署与使用

面向最终用户：如何安装 `gwf-cli`、在任意项目里初始化，并在 **Grok Build** 中直接写需求。

包名：**[gwf-cli](https://www.npmjs.com/package/gwf-cli)**（命令：`gwf`）  
仓库：https://github.com/Neko9-lab/grok_wf

---

## 1. 环境要求

| 依赖 | 版本 |
|------|------|
| Node.js | >= 18.17 |
| Python | >= 3.9（任务脚本 / 范围检查） |
| Git | 任意近期版本 |
| Grok Build | 已安装并可在终端运行 `grok` |

---

## 2. 安装 CLI（全局，装一次）

```bash
npm install -g gwf-cli --registry https://registry.npmjs.org/
gwf -V
```

> 若默认用了国内镜像、暂时装不到最新版，请加上 `--registry https://registry.npmjs.org/`。

升级：

```bash
npm install -g gwf-cli@latest --registry https://registry.npmjs.org/
# 或
gwf upgrade
```

每个**已 init 的项目**升级 CLI 后，建议再执行：

```bash
gwf update
```

---

## 3. 在任意项目中部署（每个项目一次）

```bash
cd your-project
gwf init -u 你的名字
```

### 3.1 二次确认（直接回车 = 是）

init 结束后会用**中文**询问是否开启两项（推荐都开）：

| 项 | 白话效果 |
|----|----------|
| ① 允许本项目自动加载工作流 | 打开 Grok 时自动带上任务/规范，**不用每次 `/start`** |
| ② 提交代码前检查改动范围 | 防止 AI 改到任务范围外的文件被 commit |

```text
是否开启以上两项？ [Y/n，直接回车=是]
```

其他方式：

```bash
gwf init -u 你的名字 -y              # 不提问，两项都开
gwf init -u 你的名字 --no-automations  # 都不开（需自己 /start）
```

已 init 过、想补开：

```bash
gwf enable-automations
# 或
gwf trust              # 只开「自动加载」
gwf install-hooks      # 只开「提交检查」
```

### 3.2 提交 GWF 脚手架（建议）

```bash
git add .gwf .grok AGENTS.md .gitignore
git commit -m "chore: init GWF workflow"
```

## 4. 日常使用（核心）

```bash
cd your-project
grok
# 直接说需求，例如：加一个用户登录功能
```

**推荐路径：init（回车确认）→ 打开 Grok → 说需求。**

| 阶段 | 你做什么 |
|------|----------|
| 小改 | 直接说，可不建任务 |
| 大改 | AI 会问是否建任务；确认 PRD 与**改动范围（scope）** |
| 实现中 | AI 应只在范围内改文件 |
| 提交前 | 若装了 git 钩子，越界 commit 会被拦 |
| 结束 | `/finish-work` 或让 AI 归档任务并写 journal |

### 4.1 scope（改动范围）是什么？

- **是限制**：限制「这次允许改哪些路径」，防大仓乱改。  
- **通常不用手写**：AI 起草 `scope.json`，你确认或口头说「只许改 src/xxx」。  
- 机器检查：`python .gwf/scripts/check_scope.py --strict-missing`

## 5. 常用命令

| 命令 | 作用 |
|------|------|
| `gwf init -u <name>` | 初始化 + 二次确认开启自动加载/提交检查 |
| `gwf update` | 同步项目内模板到当前 CLI 版本 |
| `gwf upgrade` | 升级全局 `gwf-cli` |
| `gwf doctor` | 检查环境、trust、hooks、版本是否一致 |
| `gwf trust` | 仅开启：打开 Grok 自动加载工作流 |
| `gwf enable-automations` | 再次弹出确认，开启上述两项 |
| `gwf install-hooks` | 仅安装 git 提交范围检查 |
| `gwf -V` | 显示版本 |

---

## 6. 故障排查

| 现象 | 处理 |
|------|------|
| 打开 Grok 像不认识 GWF | `gwf trust` 或 `gwf doctor` 看「Grok auto-inject」 |
| 没有自动上下文 | 确认在**项目根目录**开 Grok；再执行 `gwf enable-automations` |
| commit 被拒绝 | 改动超出 `scope.json`；收窄改动或更新 scope 后重试 |
| `gwf` 找不到 | `npm i -g gwf-cli` 后重开终端 |
| 镜像上没有最新版 | 加 `--registry https://registry.npmjs.org/` |

## 7. 开发者：从源码安装 / 发版

```bash
git clone https://github.com/Neko9-lab/grok_wf.git
cd grok_wf
npm install
npm run build
npm link -w gwf-cli
```

发布到 npm（需 npm 登录 + 2FA 或 granular token）：

```bash
# 在 packages/cli 升版本后
npm publish -w gwf-cli --access public --registry https://registry.npmjs.org/
```
