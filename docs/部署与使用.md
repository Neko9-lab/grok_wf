# 部署与使用

[English](./deploy-and-usage.md) | [中文](./部署与使用.md)

如何安装 **gwf-cli**、在任意项目中初始化，并在 **Grok Build** 中直接写需求。

|--|--|
| **包名** | [gwf-cli](https://www.npmjs.com/package/gwf-cli)（命令 `gwf`） |

## 1. 环境要求

| 依赖 | 版本 |
|------|------|
| Node.js | >= 18.17 |
| Python | >= 3.9（任务脚本 / 范围检查） |
| Git | 近期版本 |
| Grok Build | 终端可运行 `grok` |

---

## 2. 安装 CLI（全局，装一次）

```bash
npm install -g gwf-cli --registry https://registry.npmjs.org/
gwf -V
```

> 若默认用了国内镜像、暂时装不到最新版，请加上  
> `--registry https://registry.npmjs.org/`。

**升级：**

```bash
npm install -g gwf-cli@latest --registry https://registry.npmjs.org/
# 或
gwf upgrade
```

每个**已 init 的项目**升级 CLI 后建议再执行：

```bash
gwf update
```

---

## 3. 在项目中部署（每个项目一次）

```bash
cd your-project
gwf init -u 你的名字
```

### 3.1 二次确认（直接回车 = 是）

脚手架结束后会用**中文白话**询问是否开启两项（推荐都开）：

| 项 | 效果 |
|----|------|
| ① 允许本项目自动加载工作流 | 打开 Grok 时自动带任务/规范，**不用每次 `/start`** |
| ② 提交代码前检查改动范围 | 防止范围外改动被提交 |

```text
是否开启以上两项？ [Y/n，直接回车=是]
```

其他方式：

```bash
gwf init -u 你的名字 -y                 # 不提问，两项都开
gwf init -u 你的名字 --no-automations   # 都不开（需自己 /start）
```

已 init 过、想补开：

```bash
gwf enable-automations
# 或
gwf trust              # 只开「自动加载」
gwf install-hooks      # 只开「提交检查」
```

### 3.2 提交脚手架（建议）

```bash
git add .gwf .grok AGENTS.md .gitignore
git commit -m "chore: init GWF workflow"
```

## 4. 日常使用

```bash
cd your-project
grok
# 例如：加一个用户登录功能
```

**推荐路径：** `init`（确认）→ 打开 Grok → 说需求。

| 场景 | 你做什么 |
|------|----------|
| 小改 | 直接说，可不建任务 |
| 大改 | 确认建任务与**改动范围（scope）** |
| 实现中 | AI 应只在范围内改 |
| 提交前 | 钩子可能拦截越界 |
| 结束 | `/finish-work` 或让 AI 归档 |

### 4.1 scope（改动范围）是什么？

- **是限制：** 这次允许改哪些路径，防大仓乱改。
- **通常不用手写：** AI 起草 `scope.json`，你确认或口头指定。
- **机器检查：**

```bash
python .gwf/scripts/check_scope.py --strict-missing
```

## 5. 常用命令

| 命令 | 作用 |
|------|------|
| `gwf init -u <name>` | 初始化 + 二次确认自动化 |
| `gwf update` | 同步项目模板 |
| `gwf upgrade` | 升级全局 CLI |
| `gwf doctor` | 环境与配置检查 |
| `gwf trust` | 仅开启自动加载工作流 |
| `gwf enable-automations` | 再次确认开启两项 |
| `gwf install-hooks` | 仅安装 git 提交范围检查 |
| `gwf -V` | 显示版本 |

---

## 6. 故障排查

| 现象 | 处理 |
|------|------|
| 像不认识 GWF | `gwf trust` 或 `gwf doctor` |
| 无自动上下文 | 在**仓库根目录**打开 Grok；执行 `gwf enable-automations` |
| commit 被拒 | 超出 `scope.json`；改文件或更新 scope |
| 找不到 `gwf` | 重新全局安装；重开终端 |
| 镜像无新版 | 使用 `--registry https://registry.npmjs.org/` |

## 7. 开发与发版

```bash
git clone https://github.com/Neko9-lab/grok_wf.git
cd grok_wf
npm install
npm run build
npm link -w gwf-cli
```

发布到 npm（需登录 + 2FA 或 granular token）：

```bash
npm publish -w gwf-cli --access public --registry https://registry.npmjs.org/
```
